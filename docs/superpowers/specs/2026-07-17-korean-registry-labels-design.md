# 레지스트리 분야·분류 한국어 표시 설계

## 목표

카탈로그의 영문 분야·분류 식별자는 유지하면서, 사용자가 보는 목록·필터·모달·Atlas·상세 화면에는 일관된 한국어 표시명을 제공한다.

## 설계

- 사이트 UI 전용 중앙 번역 함수에서 분야와 분류 값을 한국어 표시명으로 변환한다.
- 필터, 정렬, 검색, URL 및 카탈로그 스키마는 기존 영문 원본 값을 계속 사용한다.
- 정보원 분야, k-skill 분류, 기술 분류를 각각 명시적 매핑으로 관리한다.
- 목록 셀과 필터 옵션, Registry 모달, Atlas, k-skill 상세, 청사진 연결 정보원에 같은 번역 함수를 적용한다.
- 아직 매핑되지 않은 새 값은 원문을 그대로 표시해 기록이 사라지거나 화면이 깨지지 않게 한다.
- ID, 기술명, 데이터 형식, 실행 환경 등 고유명사나 별도 의미 체계는 이번 변경 범위에 포함하지 않는다.

## 데이터 흐름

카탈로그 원본 값은 검색·필터용 행 데이터에 유지한다. 렌더링 시에만 번역 함수를 호출하므로 한국어 옵션을 선택해도 내부 필터 값은 기존 영문 값과 정확히 비교된다.

## 검증

- 알려진 분야·분류 값이 기대한 한국어로 변환되는지 단위 테스트한다.
- 미등록 값이 원문으로 안전하게 대체되는지 단위 테스트한다.
- 목록 필터가 한국어 옵션을 보여주면서 기존 영문 값으로 정상 필터링되는지 확인한다.
- 사이트 타입 검사, 단위 테스트, 빌드, 내부 링크 검사와 기존 Playwright 회귀 테스트를 실행한다.
